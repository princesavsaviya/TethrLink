Direct Hardware Pipeline

TetherLink bypasses your network stack entirely. Frames are encoded by GStreamer (JPEG or H.264) and pushed over a private USB tethering subnet — no router, no Wi-Fi jitter, no cloud relay. Pure point-to-point at 1–5 ms USB RTT.

terminal

Zero Cloud Dependency

No accounts. No telemetry. No remote servers. Settings persist locally to ~/.config/tetherlink/settings.json. Your stream never leaves your desk.

memory

Low Overhead

GStreamer hardware-accelerated encoding keeps CPU usage minimal. The server runs quietly in the background via a GTK4 system-tray icon while you work.

tune
tune

Configurable Output

Switch between JPEG and H.264 codecs at runtime. Change FPS and quality without dropping the stream. Configure virtual display position (above / below / left / right) and resolution (720p / 1080p / 1440p) from the GTK4 desktop app.

Settings → Stream Settings → Codec / FPS / Quality

Initialization Protocol

01

Enable USB Tethering on Android

Connect the USB cable to your Linux PC. On Android, go to Settings → Network → Hotspot & Tethering → USB Tethering and toggle it on. This creates the private 192.168.42.x subnet.

02

Start the Linux server

Install the .deb package and run tetherlink. Open the Dashboard tab and click Start Server. A virtual display is created via the Mutter ScreenCast API.

03

Open the Android app

Install the APK. It discovers the server automatically via UDP broadcast — no IP to type. The virtual display appears on the tablet within seconds.
