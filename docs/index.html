Built for speed.
Designed for privacy.

Every component of TethrLink is engineered around one principle: your display data never leaves your physical desk.

Direct Hardware Pipeline

GStreamer JPEG TCP/USB 1–5 ms RTT

Frames are encoded by GStreamer hardware-accelerated pipeline and pushed over a private USB tethering subnet. No router, no Wi-Fi jitter, no cloud relay. Pure point-to-point at 1–5 ms USB RTT with sub-50 ms end-to-end latency.

Zero Cloud Dependency

No accounts. No telemetry. No remote servers. Settings persist locally. Your stream never leaves your desk.

Auto Discovery

The Android app finds your Linux server automatically over UDP broadcast on the USB subnet. No IP address to type — just open and connect.

Low Overhead

Hardware-accelerated encoding keeps CPU usage minimal. The server runs quietly as a GTK4 system-tray icon while you work.

Mutter ScreenCast API

Virtual display creation uses the Mutter ScreenCast D-Bus API — the same mechanism GNOME uses internally, giving TethrLink a rock-solid display abstraction with no kernel module required.

# Start a 1920×1080 virtual display $ TethrLink --resolution 1920x1080 --position right ◆ Virtual display: ID=2 · 1920×1080 (right of primary) ◆ TCP server: 192.168.42.129:8080 · UDP: :5005 ◆ Stream active — 45 FPS · 37.2 Mbps · 47 ms

Initialization Protocol

Up and running
in three steps.

01

Enable USB Tethering on Android

Connect the USB cable to your Linux PC. On Android, go to Settings → Network → Hotspot & Tethering → USB Tethering. This creates the private 192.168.42.x subnet between your devices.

02

Start the Linux server

Install the .deb package and launch TethrLink. Open the Dashboard tab and click Start Server. A virtual display is created instantly via the Mutter ScreenCast API — no reboot needed.

03

Open the Android app & connect

Install the APK. The app discovers your server automatically via UDP broadcast — no IP to type. The virtual display appears on the tablet within seconds. Drag windows across like any real monitor.

FAQ

Common questions

Does it work without Wi-Fi?
Yes — that's the entire point. TethrLink uses Android's USB tethering feature which creates a private 192.168.42.x network directly over the USB cable. No Wi-Fi, no router, no internet required.
Which Linux desktop environments are supported?
TethrLink uses the Mutter ScreenCast D-Bus API, which is available on GNOME (Wayland). Ubuntu 22.04+, Fedora 37+, and any GNOME Wayland session work out of the box. X11 and non-GNOME desktops are not currently supported.
What Android version is required?
Android 8.0 (Oreo) or later. USB tethering must be supported by your device — it's available on virtually all Android phones and tablets from 2017 onwards.
What codec does TethrLink use?
TethrLink currently uses JPEG encoding via GStreamer for the video stream. JPEG keeps the pipeline simple and latency low. H.264 support is planned for a future release — subscribe to the newsletter to be notified when it lands.
Is touch input forwarded from the tablet?
Not yet — touch input forwarding is on the roadmap for a future release. Subscribe to the newsletter below to be notified when it lands.
